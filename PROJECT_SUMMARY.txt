PROJECT WALKTHROUGH — Arduino IR Sensor → MQTT Cloud Dashboard
================================================================

STEP 1: Project Setup
- Created folder: arduino_uno_connect/
- Created Python virtual environment: .venv/
- Installed pyserial (serial communication with Arduino)
- Installed paho-mqtt (MQTT client for cloud broker)

STEP 2: Detect Arduino
- Wrote arduino_detect.py to scan COM ports
- Found Arduino on COM10 (FTDI USB Serial)
- Arduino was running a temperature monitoring sketch already

STEP 3: Choose a Free MQTT Broker
- Researched free MQTT brokers (EMQX, HiveMQ, Mosquitto, MQTTHQ)
- Chose HiveMQ Cloud (Serverless Free Tier)
  - No credit card required
  - 100 connections free
  - 10 GB data per month
  - Private topics (with username/password auth)
- Signed up at console.hivemq.com
- Created Serverless cluster
- Created MQTT credential: arduino_bridge / Sathvik7
- Got connection details:
  Host: 792bbe975e73485e87707a342bce20bf.s1.eu.hivemq.cloud
  Port: 8883 (MQTT TLS) / 8884 (WebSocket WSS)

STEP 4: Wire the IR Sensor
- Connected IR Obstacle Avoidance Sensor to Arduino Uno:
  - Sensor VCC  → Arduino 5V
  - Sensor GND  → Arduino GND
  - Sensor OUT  → Arduino A2
- Used male-to-female jumper wires (no breadboard needed)
- Adjusted potentiometer on sensor to set detection range

STEP 5: Write & Upload Arduino Firmware
- Created arduino_simple_test/arduino_simple_test.ino
- Arduino reads IR sensor on pin A2 every 500ms
- Sensor logic: OUT = LOW (0) when obstacle, HIGH (1) when clear
- Arduino sends over USB Serial at 9600 baud:
    "obstacle=yes" or "obstacle=no"
- Uploaded via Arduino IDE (Board: Arduino Uno, Port: COM10)

STEP 6: Write the Python Bridge (bridge.py)
- Auto-detects Arduino COM port
- Opens serial connection at 9600 baud
- Connects to HiveMQ Cloud with TLS on port 8883
- Reads each line from Arduino
- Publishes to topic: arduino/sensor/data
- Payload format: JSON {"raw": "...", "timestamp": ...}

STEP 7: Write the Subscriber (subscriber.py)
- Connects to HiveMQ Cloud with TLS on port 8883
- Subscribes to topic: arduino/sensor/#
- Prints every received message with topic + payload

STEP 8: Write the Web Dashboard (dashboard.html)
- Single HTML file (no web server required)
- Uses MQTT.js library to connect via WebSocket (WSS on port 8884)
- Connects directly to HiveMQ Cloud from the browser
- Features:
  - Status ring: green ✓ = clear, red ⚠ pulsing = obstacle detected
  - Bar chart: rolling 15-minute detection rate (detections/minute)
  - Collapsible event log (click to show/hide)
  - Connection status indicator

STEP 9: Run the System
- Terminal 1: python bridge.py      (Arduino → Cloud)
- Terminal 2: python subscriber.py  (Cloud → terminal)
- Browser: open dashboard.html      (Cloud → browser)

DATA FLOW
  Arduino (IR sensor)

      ↓ MQTT (TLS port 8883)
  HiveMQ Cloud
      ↙             ↘
  subscriber.py    dashboard.html
